# 출처와 재현

## 공식 기준선

접근일은 2026-07-13이다. 총론·중학교·고등학교는 국가교육위원회 고시 제2026-1호의 별책 1·3·4를 사용한다. 교과·선택·전문교과는 제2024-3호가 다시 발행한 별책을 우선하고, 이후 대체되지 않은 별책은 [교육부 고시 제2022-33호](https://www.moe.go.kr/boardCnts/viewRenew.do?boardID=141&boardSeq=93458&lev=0)를 사용한다. 창의적 체험활동은 제2022-33호 별책 40을 사용한다. 현행 고시와 별책 대체 여부는 [NCIC 국가교육과정정보센터](https://ncic.re.kr/?m=4)에서 대조한다.

선택된 38개 파일의 공식 첨부 번호, 원래 파일명, 고시, 선택 이유는 `sources/official/source-catalog.json`에 있다. 실제 내려받은 바이트 수·SHA-256·PDF 페이지·추출 텍스트 파일은 `source-receipts.json`에 고정했다. 총 수집 바이트는 210,511,860이다.

## 생성 과정

1. NCIC 세션과 CSRF 토큰을 통해 각 공식 첨부를 내려받는다.
2. PDF 헤더·바이트 수·SHA-256·페이지 수를 기록하고 `pdftotext -layout`으로 로컬 텍스트를 만든다.
3. 학교급 코드 패턴과 별책 범위를 이용해 성취기준 코드를 탐지한다.
4. 별책→교과군→과목→영역→성취기준 계층과 PDF locator를 만든다.
5. 공식 문장 자체는 제외하고 `mechanical-derivative`로 표시한 초안 요약·세부 주제·증거·평가 질문을 생성한다. 중학교는 초등 학습지도의 과목별 분해 밀도를 참고한 `middle-subject-facet-decomposition-v1` 정책으로 기준당 2~5개 후보를 만들고, 기존 핵심 주제 ID를 보존한다.
6. `build-learning-relations.mjs`가 `scripts/lib/official-relation-specs/`의 과목별 사양(46개 모듈, 전건 공식 문서 인쇄 쪽번호 근거)만을 소비해 중·고 관계, 중→고 bridge, 초등→중 bridge를 결정적으로 생성한다. 문서 배열 순서나 저장소 추론 기반 관계는 생성하지 않는다.
7. 모든 관계의 참조·중복·DAG와 검토 레코드 포함 여부를 검사한 뒤 JSON-LD/Turtle과 UI 인덱스를 만든다.
8. 모든 배포 파일의 바이트 수와 SHA-256을 manifest에 기록한다.

완전 재현 명령은 `bun run rebuild:all`이다. 네트워크 없이 저장소에 고정된 후보 릴리스를 검증하는 명령은 `bun run verify`다.

## 해석 한계

성취기준 코드는 공식 PDF에서 확인했지만, 과목 섹션 자동 판별·영역 제목·요약·세부 주제·증거·평가 질문·경로는 저장소 생성물이다. 이 항목은 `candidate` 또는 `mechanical-derivative`로 표시한다. v0.5부터 학습 관계는 공식 문서(내용 체계표 학년 진행, 성취기준 해설·적용 시 고려 사항, 교과 설계의 명시적 연계 문장)가 직접 뒷받침하는 것만 수록하며, 문서 배열 기반 탐색 순서는 삭제했다. 공식 문장이 학습 영역 전체를 지목한 경우 해당 영역 성취기준으로 기계적으로 전개했고 이는 과목별 검토 문서에 기록되어 있다. 교과 및 직업계 전문가의 의미 정밀화와 문서별 권리 검토는 후속 거버넌스 작업이며 자동 게이트 통과로 대체되지 않는다.

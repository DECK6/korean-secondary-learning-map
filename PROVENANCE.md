# 출처와 재현

## 공식 기준선

접근일은 2026-07-11이다. 총론·중학교·고등학교는 국가교육위원회 고시 제2026-1호의 별책 1·3·4를 사용한다. 교과·선택·전문교과는 제2024-3호가 다시 발행한 별책을 우선하고, 이후 대체되지 않은 별책은 교육부 고시 제2022-33호를 사용한다. 창의적 체험활동은 제2022-33호 별책 40을 사용한다.

선택된 38개 파일의 공식 첨부 번호, 원래 파일명, 고시, 선택 이유는 `sources/official/source-catalog.json`에 있다. 실제 내려받은 바이트 수·SHA-256·PDF 페이지·추출 텍스트 파일은 `source-receipts.json`에 고정했다. 총 수집 바이트는 210,511,860이다.

## 생성 과정

1. NCIC 세션과 CSRF 토큰을 통해 각 공식 첨부를 내려받는다.
2. PDF 헤더·바이트 수·SHA-256·페이지 수를 기록하고 `pdftotext -layout`으로 로컬 텍스트를 만든다.
3. 학교급 코드 패턴과 별책 범위를 이용해 성취기준 코드를 탐지한다.
4. 별책→교과군→과목→영역→성취기준 계층과 PDF locator를 만든다.
5. 공식 문장 자체는 제외하고 `mechanical-derivative`로 표시한 초안 요약·세부 주제·증거·평가 질문을 생성한다.
6. 중학교·고등학교 데이터와 중→고 bridge를 독립 검증한 뒤 JSON-LD/Turtle과 UI 인덱스를 만든다.
7. 모든 배포 파일의 바이트 수와 SHA-256을 manifest에 기록한다.

완전 재현 명령은 `bun run rebuild:all`이다. 네트워크 없이 저장소에 고정된 후보 릴리스를 검증하는 명령은 `bun run verify`다.

## 해석 한계

성취기준 코드는 공식 PDF에서 확인했지만, 과목 섹션 자동 판별·영역 제목·요약·세부 주제·증거·평가 질문·전이·과목 관계·경로는 저장소 생성물이다. 이 항목은 `candidate`, `mechanical-derivative` 또는 `repository-authored`로 표시한다. 코드의 배열 순서는 교육적 선수 관계의 근거가 아니므로 해당 방식으로 만들었던 관계 50,565건은 v0.2에서 모두 제거했다. 현재 주제 선수 관계는 0건이며, 새 관계는 공식 근거나 교과 전문가 검토 레코드가 있을 때만 추가한다. 교과 및 직업계 전문가의 내용 검토는 후속 거버넌스 작업이며 자동 게이트 통과로 대체되지 않는다.

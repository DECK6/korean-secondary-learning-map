# v0.5.0-candidate 릴리스 보고서

릴리스 기준일: 2026-07-17

## 변경 목적

v0.4는 공식 근거 관계 101건 위에 문서 배열 기반 탐색 순서 52,191건을 함께 수록했다. 소유자 결정에 따라 v0.5는 **공식 문서가 직접 뒷받침하는 관계만** 제품에 남긴다. 이 학습지도의 목적은 공식 교육과정 안에서의 학습 추적이며, 공식 범위를 벗어난 저장소 추론 관계는 그 목적에 부합하지 않기 때문이다.

## 핵심 변경

1. **AI 작성(repository-authored) 관계 전면 삭제**: 중학교 2,125건, 고등학교 49,990건, 학교급 bridge 과정 수준 37건. builder에서 생성 경로 자체를 제거했고, 스키마가 `repository-authored` 학습 관계를 거부한다.
2. **공식 문서 채굴을 전 교과로 확장**: 수학 파일럿(내용 체계표 학년 진행 + 성취기준 해설 인용)을 중학교 전 과목과 고등학교 전 교과(일반·교양·계열·직업계 전문교과 별책 5~39)에 적용했다. 과목별 사양은 `scripts/lib/official-relation-specs/`의 46개 데이터 모듈로 고정하며, 각 모듈은 과목별 검토 문서(`docs/reviews/2026-07-17-*.md`)와 짝을 이룬다.

## 채굴 증거 규칙

- C계층: 내용 체계표의 학년(군) 진행이 같은 갈래의 내용 요소 선후를 직접 보여줄 때 (중학교 내부는 1~3학년 단일 학년군이라 대부분 0건 — 수학만 예외).
- D계층: 성취기준 해설·적용 시 고려 사항·과목 성격이 선행 학습을 명시 지목("~을 토대로", "~을 바탕으로", "~을 이수한 후")할 때.
- 공식 문장이 학습 영역 전체를 지목한 경우 해당 영역 성취기준으로 기계적으로 전개하고 검토 문서에 전개 방식을 기록.
- 직업계 전문교과는 편성 기준·해설의 명시 문장만 인정, NCS 능력단위 나열 순서는 근거로 사용하지 않음.
- 근거가 없으면 빈 배열. 문서 배열 순서·교육적 추론 금지.

## 남은 한계

- `internal-reviewed`는 저장소 내부 검토(소유자 승인 범위)이며 교과 전문가·학교 현장 승인이 아니다.
- 세부 주제(facet) 분해·요약·증거·평가 질문은 여전히 저장소 생성물(`mechanical-derivative`, `candidate`)이다. 공식 근거 원칙은 관계에 적용되었다.
- 영역 지목 문장의 성취기준 전개는 기계적 규칙이므로 교과 전문가의 표본 검토가 필요하다.
- 공식 PDF의 문서별 재사용 권리 검토는 `HOLD` 상태다.
